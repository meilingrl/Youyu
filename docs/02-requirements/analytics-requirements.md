# 数据分析需求

## 1. 目标

本子系统同时服务于：

- 课程中“数据采集、数据清洗、数据分析、数据可视化”的要求
- 平台后台运营与店铺经营的真实分析需求

当前阶段不追求复杂 BI 系统，而追求“业务数据真实、指标口径清晰、图表可解释”。

## 2. 数据采集

必须支持：

- 用户数据采集
- 商品数据采集
- 订单数据采集
- 支付数据采集
- 评价数据采集
- 举报数据采集

应支持：

- 浏览记录采集
- 收藏行为采集
- 店铺访问采集

说明：

- 初期可直接从业务数据库采集
- 后续可扩展文件导入或独立分析表

## 3. 数据清洗

必须支持：

- 去重
- 空值处理
- 字段标准化
- 状态归一化
- 时间维度标准化

应支持：

- 异常数据识别
- 脏数据隔离
- 无效行为过滤

## 4. 平台分析指标

### 4.1 平台级指标

必须支持：

- 销售额趋势
- 订单量趋势
- 热销商品排行
- 订单状态分布
- 分类销量分布
- 用户消费偏好分布

应支持：

- 复购分析
- 校园场景维度分析
- 专业维度分析
- 用户活跃度分析

### 4.2 用户侧洞察指标

应支持：

- 累计消费金额
- 累计购买件数
- 最近购买记录
- 最近浏览记录
- 收藏偏好
- 常购分类

### 4.3 店铺侧经营指标

应支持：

- 本月销售额
- 本月订单数
- 热销商品
- 浏览量
- 收藏量
- 复购用户数

## 5. 数据可视化

必须支持：

- 统计卡片
- 折线图
- 柱状图
- 饼图或环图
- 后台运营看板

应支持：

- 用户中心轻量统计面板
- 店铺中心轻量经营面板

要求：

- 图表必须基于真实业务数据或明确可替换的真实查询接口
- 指标名称、时间范围、筛选条件必须可解释
- 不得出现无法说明口径的“装饰性图表”

## 6. 当前阶段交付策略

当前阶段采用“双层交付”：

### 第一层：结构先行

先交付：

- 指标定义
- 页面壳子
- 接口预留
- 分析实体或查询预留

### 第二层：真实数据接入

在主业务持久化完成后接入：

- 真实聚合查询
- 真实时间范围筛选
- 真实排序与排行

## 7. 与平台业务的价值

数据分析能力应直接服务于以下业务问题：

- 识别开学季、考试周、毕业季等高峰场景
- 识别热门分类与专业需求
- 识别优质店铺与潜力卖家
- 支撑商品推荐与活动运营
- 让用户和店主理解自己的交易与消费行为
