# 用户偏好与个人洞察需求

## 1. 目标

本文档用于定义两类低耦合但高价值的增强能力：

- 用户偏好设置
- 用户个人洞察与店铺经营洞察

这两类能力允许与主业务持久化并行预研，但其真实数据基础必须建立在稳定的交易、浏览、收藏和认证数据之上。

## 2. 用户偏好设置

### 2.1 界面偏好

应支持：

- 昼夜模式切换
- 主题色选择
- 首页卡片或列表展示偏好
- 商品列表默认排序偏好

约束：

- 偏好设置不能破坏整体视觉统一性
- 主题能力必须建立在统一设计变量之上实现

### 2.2 交易与账户默认设置

应支持：

- 默认收货地址
- 默认交付方式
- 默认支付方式
- 默认筛选条件
- 是否接收某类提醒

### 2.3 当前阶段交付策略

当前阶段建议：

- 先建 `用户设置` 模型
- 先建设置页与基础接口
- 先支持最核心的默认地址、默认交付方式、主题切换
- 更复杂的通知、推荐、个性化排序后置

## 3. 用户个人洞察

### 3.1 个人消费洞察

应支持：

- 累计消费金额
- 累计购买件数
- 最近购买记录
- 最近浏览记录
- 收藏偏好分类
- 高频浏览分类

### 3.2 卖家与店主洞察

应支持：

- 本月销售额
- 本月订单数
- 热销商品
- 最近成交记录
- 浏览量
- 收藏量

## 4. 页面形态建议

### 4.1 用户中心

建议新增或预留：

- 个人统计卡片区
- 最近浏览区
- 最近购买区
- 偏好设置入口

### 4.2 店铺中心

建议新增或预留：

- 本月销售额卡片
- 本月订单数卡片
- 热销商品榜
- 浏览与收藏摘要

## 5. 当前阶段实现边界

当前阶段允许：

- 页面壳子
- 指标定义
- 接口预留
- 基础表结构预留

当前阶段不要求：

- 复杂推荐算法
- 复杂个性化排序
- 复杂用户画像模型
- 大规模实时分析
