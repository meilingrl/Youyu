# Youyu UI/UX 设计宪法

## 1. 文档目的

本文定义 `Youyu` 下一轮前端大规模 UI/UX 重构的上层设计规则。

它不是页面实现任务，不是组件清单，也不是参考网站摘抄。它的作用是把当前讨论过的审美方向、交互原则、动效规则、响应式要求和 AI 执行边界沉淀成一份可复用的设计约束。

当前只定义从宏观到微观的设计语言。具体页面结构暂时留空，后续单独讨论。

## 2. 产品气质定位

`Youyu` 应该是一个可信、温暖、愿意逛、容易学会的校园交易平台。

前台应该让用户愿意浏览、发现、比较和交易。后台应该让管理者快速扫描、审核、治理和决策。

界面不应该像：

- 通用后台模板。
- 课程作业式功能堆叠。
- 默认 AI 生成的 SaaS 落地页。
- 冰冷的科技蓝管理系统。
- 带图片的商品数据库。

### 2.1 核心体验关键词

- 温暖。
- 清爽。
- 有编辑感。
- 愿意逛。
- 可信。
- 有学生产品气质。
- 轻高级。
- 响应式。
- 易学习。

### 2.2 参考方向

参考站只提供设计信号，不直接整站照抄。

| 参考源 | 借鉴重点 | 适用范围 |
| --- | --- | --- |
| Etsy | 温暖电商感、商品卡片、浏览节奏、柔和交易氛围 | 前台市场页面 |

## 3. 设计哲学

### 3.1 一屏一个主意图

每个可见屏幕应该只有一个最重要的意图。用户进入页面后，要能快速明白这里是做什么的，以及下一步最自然应该做什么。

页面可以支持多个任务，但不能把所有任务都用同等视觉权重摆出来。

### 3.2 前台先浏览，后台先操作

前台优先让人愿意逛。它应该支持发现、比较、收藏、进入详情和完成交易。

后台优先让人省力。它应该支持扫描、筛选、审核、对比、处理状态和重复操作。

### 3.3 讲故事，不堆功能

重要页面应该有连续引导感：

1. 先让用户理解这个区域是什么。
2. 再给出最重要的入口。
3. 然后展示可探索内容。
4. 接着补充信任或保障信息。
5. 最后自然引导用户进入下一步。

可交互控件必须明显可用，但页面不能因此变吵。目标是让用户可以顺着页面一路点下去，而不是先学习一张功能地图。

## 4. 视觉系统

### 4.1 色彩方向

第一版主主题学习 Etsy 的温暖生活感，不使用通用科技蓝作为默认品牌气质。

推荐方向：

- 背景使用暖白、米白、纸感浅色。
- 主文字使用炭黑或柔和黑，不使用纯黑压迫页面。
- 次级文字使用暖灰。
- 强调色使用赤陶、暖橙、烘焙色一类的温暖色。
- 辅助色可以使用低饱和苔绿、烟粉、灰棕、深酒红等。

蓝色不是禁用色，但不作为第一版默认主色。紫色也不是禁用色，真正要避免的是没有设计理由的默认紫蓝渐变，因为它会让产品看起来像普通 AI 模板站。

### 4.2 色彩比例

色彩用来建立层级，不用来填满页面。

| 层级 | 建议占比 |
| --- | --- |
| 暖中性背景与基础表面 | 70% |
| 白色或轻抬升表面 | 20% |
| 强调色与状态色 | 10% |

规则：

- 大部分界面保持温暖中性。
- 强调色只用于主操作、选中状态、信任提示、精选模块。
- 状态色保持易懂，不故意创新。
- 同一视口不出现多组互相抢戏的高饱和颜色。

### 4.3 字体与文本

文字应该清楚、克制、稍微有编辑感。

规则：

- 标题短而具体。
- 正文友好直接。
- 功能页不要写成营销口号集合。
- 用户可见文本不要暴露实现术语。
- 说明文本控制行宽，不铺满整行。
- 优先用字号、字重、间距、位置建立层级，再用颜色辅助。

建议层级：

| 类型 | 作用 |
| --- | --- |
| 展示标题 | 只用于真正的 hero 或大型介绍区域 |
| 页面标题 | 说明页面主要意图 |
| 分区标题 | 说明一个内容区域或决策区域 |
| 卡片标题 | 商品、店铺、任务、选项名称 |
| 元信息 | 时间、分类、状态、数量、辅助说明 |

### 4.4 间距

间距要带来呼吸感和确定感。高密度只用于后台或重复操作场景。

| 场景 | 建议间距 |
| --- | --- |
| 大区块之间 | 56px 到 80px |
| 中型内容组之间 | 32px 到 40px |
| 卡片内部 | 16px 到 24px |
| 行内控件之间 | 8px 到 16px |
| 桌面端页面左右边距 | 32px 到 48px |
| 平板端页面左右边距 | 24px |
| 移动端页面左右边距 | 16px |

页面不要变成一堆互不相干的浮动卡片。页面分区可以是开放布局或全宽色带。卡片主要用于重复项、真实面板、商品/店铺预览、弹窗和工具容器。

### 4.5 圆角与层次

圆角和立体感可以大胆使用，但必须服务层级。

| 元素 | 建议圆角 |
| --- | --- |
| 大型精选卡片 | 24px 到 32px |
| 商品或店铺卡片 | 18px 到 24px |
| 浮动面板 | 20px 到 28px |
| 输入框或搜索壳 | 999px 或胶囊形 |
| 小标签 chip | 999px |
| 后台表格或面板 | 8px 到 16px |

层次来源优先级：

1. 留白。
2. 边框。
3. 阴影。
4. 背景层级。
5. 毛玻璃。

不要让单一视觉技巧承担全部高级感。

## 5. 毛玻璃与浮层

毛玻璃允许使用，而且可以成为 `Youyu` 的识别点之一。

规则不是少用毛玻璃，而是毛玻璃必须有权力分配。它应该出现在真正浮在内容之上的区域，或者临时承载注意力的区域。

推荐使用位置：

- 顶部导航。
- 首页主搜索入口。
- 浮动筛选条。
- 弹窗。
- 抽屉。
- 登录/注册面板。
- 精选内容的信息浮层。
- 紧凑快捷操作面板。

不推荐使用位置：

- 所有商品卡片。
- 所有列表项。
- 大面积后台表格。
- 长列表主区域。

毛玻璃层级：

1. 导航和搜索可以使用最强毛玻璃。
2. 弹窗和抽屉使用中等毛玻璃。
3. 小型覆盖层使用轻毛玻璃。
4. 重复型商品卡片通常更适合实体表面。
5. 后台表格保持扁平、清晰、可读。

## 6. 交互系统

### 6.1 动作层级

不是所有可点击元素都要长得像按钮。减少重复按钮形状，是提升高级感和降低页面压力的重要手段。

| 层级 | 视觉形式 | 使用场景 |
| --- | --- | --- |
| 主操作 | 真按钮，最强填充或最高视觉权重 | 购买、发布、确认、登录、提交、继续 |
| 次操作 | 轻按钮、弱底色、描边按钮 | 有用但不主导的操作 |
| 第三级操作 | 文字交互、下划线、箭头、颜色变化 | 查看更多、进入详情、低风险跳转 |
| 行内操作 | 嵌入文本或卡片结构 | 局部上下文动作 |
| 图标操作 | 图标按钮，必须有可访问标签或 tooltip | 高频工具型操作 |

文字直接可交互是允许的，尤其适合减轻页面按钮密度。它必须有 hover、focus、active 等清晰反馈。

### 6.2 按钮方向

按钮可以学习 Apple 的清爽和确定感：

- 形状明确。
- 文案简短。
- 对比强但不吵。
- hover 与 press 平滑。
- 一个视口内主按钮数量少。

规则：

- 不要让每张卡片都有醒目的大按钮。
- 探索类入口优先使用卡片点击、文字链接或轻交互。
- 主按钮主要服务交易、提交、发布、登录、确认。
- 按钮文本要短，直接表达动作。

### 6.3 可点击性

能交互的控件一定要能看出来可以交互。

可用信号包括：

- 指针光标。
- hover 抬升。
- 阴影变化。
- 文字下划线动画。
- 箭头位移。
- 背景填充。
- 边框增强。
- 图标轻移。
- focus ring。

不要为了极简而隐藏交互。

## 7. 搜索与筛选

搜索和筛选重点学习 Airbnb 的探索式入口。

### 7.1 Airbnb 式搜索模式

搜索入口不是普通表单，而是用户开始浏览旅程的入口。

应该具备：

- 默认态低压、清爽、明显可点击。
- 外层容器通常是胶囊形或大圆角软壳。
- 内部可以按意图分段，例如关键词、分类、类型、交易方式。
- 聚焦时整个容器抬升，通过阴影、缩放或背景层级变强。
- 当前激活分段更突出，其他分段略微退后。
- 搜索动作可见，但不吵。
- 建议、历史记录或高级筛选像同一个表面的延伸，而不是突然掉下来的表单。
- 展开和收起要有重量感，速度要有明显加速和减速。

### 7.2 搜索状态

| 状态 | 必须表现 |
| --- | --- |
| 默认 | 清爽、明确、可点击 |
| hover | 边框或阴影轻微增强 |
| focus | 容器抬升，层级增强，必要时出现建议 |
| 已选筛选 | chip 或分段既有颜色变化，也有层级变化 |
| 加载 | 轻量进度或骨架，不整页阻塞 |
| 无结果 | 友好解释，并给出下一步 |

### 7.3 筛选规则

筛选要帮助探索，而不是变成后台检索表单。

- 标签短。
- 优先使用 chip、分段控件、轻菜单。
- 选中状态必须明显。
- 移动端不要一次展示过多筛选。
- 移动端筛选可以进入底部面板或抽屉。

## 8. 卡片系统

卡片结合 Etsy 的商品信息组织和 Patagonia 的触感式 hover。

### 8.1 商品卡片

推荐信息顺序：

1. 图片。
2. 分类或小标签。
3. 商品标题。
4. 价格。
5. 小型信任或活跃信号。
6. 可选操作或文字链接。

规则：

- 商品卡片要适合浏览和扫描。
- 不要塞入所有业务字段。
- 商品图可以真实而不完美，但裁切、比例、容器必须稳定。
- 同一行卡片高度要尽量一致。

### 8.2 店铺卡片

推荐信息顺序：

1. 店铺视觉或缩写。
2. 店铺名。
3. 短定位文案。
4. 评分、交易量或信任信号。
5. 精选商品预览或进入动作。

### 8.3 卡片交互

默认 hover：

- 上移 2px 到 4px。
- 阴影略增强。
- 边框或表面对比增强。
- 图片可在容器内轻微放大。
- 文本布局保持稳定。

禁止剧烈弹跳、旋转和大幅缩放。

## 9. 图片与媒体

图片可以承担更多设计表现力。精选模块和宣传模块可以使用生成图，但风格必须统一。

### 9.1 图片规则

- 商品图可以保留用户生成内容的真实感。
- 精选图必须统一光线、色温、构图密度和风格。
- 使用稳定比例。
- 使用有意图的裁切。
- 不要在同一模块里随机混合插画、3D、摄影和海报风。
- 避免廉价促销海报感。

### 9.2 Apple 式精选展示

Apple 值得学习的不是单纯大图，而是：

- 强横向滚动节奏。
- 主次卡片清楚。
- 卡片间距控制稳定。
- CTA 干净。
- 运动在接近停点时漂亮减速。

`Youyu` 可以把这种方式用于精选商品、精选店铺、校园场景、主题集合。实现时不能依赖真实商品图都足够漂亮，需要通过容器、裁切和生成图保证质感。

## 10. 动效系统

动效是这轮设计的重要部分。目标是昂贵感、清晰、顺滑、有引导性。

### 10.1 动效目标

动效应该让界面感觉：

- 顺。
- 有引导。
- 有响应。
- 有重量。
- 易跟随。

不要为了炫而动。

### 10.2 时间尺度

| 动效类型 | 时长 |
| --- | --- |
| 微交互 | 140ms 到 180ms |
| 普通过渡 | 220ms 到 280ms |
| 搜索/筛选展开 | 280ms 到 360ms |
| 精选 carousel 或大型面板切换 | 360ms 到 480ms |

### 10.3 速度曲线

速度变化比位移幅度更重要。

规则：

- 进入时起步明确，接近终点明显减速。
- 离开时更快、更干净。
- 大内容移动要有重量感。
- 搜索展开要像一个表面被打开，不像普通下拉框突然出现。
- 横向精选滚动要有受控的惯性和减速。

优先动画属性：

- `transform`
- `opacity`
- `filter`
- `box-shadow`
- `background-color`
- `border-color`

尽量避免频繁动画布局属性。

### 10.4 动效边界

不是每个区块都要动。动效只在它能说明层级、焦点、连续性或可点击性时使用。

避免：

- 到处视差。
- 大幅弹跳。
- 大幅缩放。
- 装饰性旋转。
- 多个模块同时入场。
- 拖慢高频操作的动效。

## 11. 响应式设计

响应式是硬要求。产品不能只是缩小后还能看，而是要在不同屏幕上重新组织体验。

### 11.1 断点哲学

桌面端：

- 完整浏览体验。
- 更丰富布局。
- 更多可见筛选。
- 更宽商品网格。

平板端：

- 减少列数。
- 仍支持对比。
- 筛选可以压缩成 chip 或面板。

移动端：

- 一次只给一条清晰路径。
- 搜索入口必须强。
- 可使用底部导航、底部面板或抽屉。
- 触控目标舒适。
- 卡片不被挤碎。

### 11.2 响应式规则

- 点击目标至少 40px 高。
- 搜索和筛选在移动端必须适合单手操作。
- 重要操作保持可达。
- 长按钮文案要能换行或缩短。
- 商品卡片需要稳定比例和可预测高度。
- 除非模块本身就是横向滚动，否则避免横向溢出。
- 桌面多列区块在移动端变成单列或横向模块。

## 12. 前台与后台的风格边界

### 12.1 前台

前台可以更温暖、更视觉化、更探索、更有动效。

它要支持：

- 浏览。
- 搜索。
- 商品发现。
- 店铺发现。
- 信任建立。
- 购买决策。
- 学生友好引导。

### 12.2 后台

后台更安静、更操作化。

它要支持：

- 扫描。
- 筛选。
- 审核。
- 对比。
- 重复操作。
- 快速理解状态。

后台规则：

- 毛玻璃更少。
- 大圆角更少。
- 动效更少。
- 表格更清楚。
- 状态标签更稳定。
- 信息层级更强。
- 主要参考 Stripe Dashboard 和 Linear。

## 13. 页面结构

页面结构标准见 `frontend-information-architecture.md`。

当前结论：

- 首页轻量化，不承担完整商品列表职责。
- 一级入口采用首页、探索、消息、交易、我的。
- 探索页承载商品、搜索、筛选、精选店铺、热搜热门。
- 商品详情和店铺详情是探索页之后的核心承接页。
- 交易作为一级能力，收纳购物车、订单、售后、支付状态、评价。
- 消息作为一级能力，支持用户、店铺、客服、群聊等沟通对象。
- 个人主页与设置中心分离。
- 店铺是店主的对外主体，店主能力属于前台。
- 后台只表示系统管理员，并补充客服与消息治理。

## 14. AI 执行规则

后续 AI 进行 UI 工作时必须遵守以下约束。

### 14.1 必须做

- 第一版主题保持温暖，学习 Etsy 的生活感。
- 保持前端、后端、数据库分离。
- 默认复用现有 Vue、Pinia、Router、API、Element Plus 模式。
- 优先建设 token 和共享组件，再写页面局部样式。
- 所有可交互控件必须显得可交互。
- 响应式行为从一开始就设计，不作为收尾补丁。
- 重要视觉变更必须用真实浏览器验证。
- 前台和后台的视觉强度必须区分。

### 14.2 禁止做

- 不要默认使用科技蓝作为品牌主色。
- 不要把紫色当禁用色，但不要使用默认 AI 紫蓝渐变作为主风格。
- 不要把所有动作都做成大按钮。
- 不要为了极简隐藏交互。
- 不要没有层级地到处使用毛玻璃。
- 不要让每个页面都像 landing page。
- 不要在商品卡片里塞满字段。
- 不要把移动端做成压缩桌面端。
- 不要随意引入新的 UI 框架。
- 页面叙事没有定稿前，不要重写页面结构。
