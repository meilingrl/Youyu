# Glossary

## Core Planning Terms

- Development process
  - Repository-level delivery and documentation workflow rules.
- Stage roadmap
  - Phase and stage progression across the project lifecycle.
- Feature roadmap
  - Ordered list of feature workstreams and their dependencies.
- Task
  - A concrete, bounded implementation item.
- Cross-cutting task
  - A task that addresses a concern spanning multiple modules or features.

## Task Status Terms

- Draft
  - Not yet approved or not yet execution-ready.
- Active
  - Approved and currently relevant for implementation.
- Archived
  - Completed or superseded but preserved for traceability.

## Architecture Terms

- API contract
  - The expected request/response behavior captured in code and `.http` files.
- ADR
  - Architecture Decision Record used for significant technical decisions.
- Runtime baseline
  - The database, framework, and environment assumptions used by the current implementation.

## Governance Terms

- Single source of truth
  - The one document class that owns a fact or rule.
- Completion notes
  - The final recorded summary of what a task actually delivered, including deviations if needed.
