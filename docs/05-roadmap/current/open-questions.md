# 当前待决问题

本文档只记录仍会影响后续开发决策的问题。

## 1. 聊天 MVP 的最小边界

需要确认：

- 是否只做会话列表、消息列表、发送消息、轮询刷新
- 是否首版支持围绕商品和店铺发起会话
- 是否首版就引入未读数和最后消息摘要

影响：

- 直接决定聊天任务拆分、数据表设计和前端页面范围

## 2. 平台调解与举报系统的关系

需要确认：

- 调解是否建立在聊天会话之上
- 调解与现有 `report` 是并行入口还是升级关系
- 管理端是复用现有治理页还是单开一条流程

影响：

- 决定平台调解的任务规格与数据模型边界

## 3. 热搜 P3 的目标界定

需要确认：

- P3 先追求功能补全，还是同时给出可接受的聚合性能基线
- suggestion 是仅前缀匹配，还是引入更复杂的策略

影响：

- 决定 P3 是轻量功能任务，还是功能与性能联合任务

## 4. 是否建立正式 API 规范流程

需要确认：

- 后续是否要求核心模块产出正式 API 说明
- 如果要做，采用模板化手写文档，还是引入 OpenAPI 导出

影响：

- 决定接口协作资产的维护方式与文档增量成本
