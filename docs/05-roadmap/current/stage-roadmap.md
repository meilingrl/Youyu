# 当前阶段路线图

本文件只记录“当前阶段正在做什么、下一步做什么、当前风险是什么”。

它不是项目历史总结，也不负责记录已经完成的早期阶段细节。

## 当前阶段

当前阶段定义为：

- 核心交易链路已完成
- 热搜、评价、推荐等主要扩展能力已基本落地
- 聊天 MVP 已完成
- 消息中心与交易中心 UI/UX 第二轮已完成
- 营销 MVP 第一阶段已完成
- 项目进入“功能收口 + 上线准备”并行阶段

这一阶段的重点是：

1. 按 `feature-roadmap.md` 补齐功能断点和后续产品能力
2. 按 `launch-preparation-roadmap.md` 完成安全、合规、基础设施、部署、监控和验收
3. 保持功能任务与上线准备任务分离，跨域任务通过依赖关系衔接

**功能路线**：`feature-roadmap.md` 提供 F0-F6 功能完成顺序。
**上线路线**：`launch-preparation-roadmap.md` 提供 L0-L7 上线准备路线。

## 当前优先级

### 功能 P0

- **契约断点补齐**：收藏、分类、地址完整管理、评价分布、HTTP/API 对齐
- **支付展示口径修正**：用户可见页面不暴露内部 mock/test 文案

### 功能 P1

- **客服、售后与交易状态深化**：客服队列、工单状态、退款协商、物流/线下交付细节、通知收口
- **支付功能升级**：网关抽象、失败/取消/超时、支付宝沙箱、退款一致性

### 功能 P1/P2

- **数据采集、分析与可视化**：行为采集、指标 API、后台/用户/店铺图表
- **个性化与偏好能力**：主题、通知、隐私、默认流程偏好、推荐个性化

### 功能 P2/P3

- **运营、营销与后台扩展**：高级活动规则、批量操作、系统配置、店铺运营增强
- **全局 UI/UX 与移动端收口**：设置中心、非交易主流程移动适配、状态统一、最后一轮包体积治理

### 上线 P0

- **安全加固**：HTTPS、JWT 密钥、CORS、输入验证、SQL 注入防护、权限回归
- **隐私合规**：隐私政策、用户协议、Cookie 同意、用户同意日志、数据导出、账号删除
- **基础设施与部署**：数据库连接池、日志、健康检查、备份、Docker、Nginx、CI/CD、回滚

### 上线 P1

- **性能容量**：性能基线、Redis 缓存候选、限流、HTTP 缓存、压测
- **监控运维**：指标、告警、日志检索、runbook、应急预案

## 上线入口条件

只有在以下条件满足后，才适合正式上线：

- `feature-roadmap.md` 中纳入上线候选的功能阶段已完成并通过验证
- `launch-preparation-roadmap.md` 中 L1/L2/L3/L5/L7 的 P0 项完成
- 性能测试达标（500 并发用户，p95 <500ms，错误率 <1%）
- 安全扫描无高危漏洞
- 备份恢复流程经过演练验证
- 运维文档和应急预案完整
- 合规性检查通过（ICP 备案如需要）

## 当前风险

- **范围混淆风险**：功能收口和上线准备必须拆任务推进，避免一个任务同时改产品能力和部署安全
- **功能契约风险**：收藏、分类、地址等已有入口若不先补齐，会影响后续 smoke 和验收
- **支付升级风险**：真实网关和退款一致性会影响订单状态机、安全验收和客服售后
- **统计口径风险**：数据可视化必须先统一采集和指标定义，否则图表会反复返工
- **性能优化风险**：Redis 缓存失效策略复杂，需要充分测试
- **数据迁移风险**：图片迁移到 OSS 需要无损且可回滚
- **合规性时间**：ICP 备案需要 20-30 个工作日，需提前启动
