# 当前阶段路线图

本文件只记录“当前阶段正在做什么、下一步做什么、当前风险是什么”。

它不是项目历史总结，也不负责记录已经完成的早期阶段细节。

## 当前阶段

当前阶段定义为：

- 核心交易链路已完成
- 热搜、评价、推荐等主要扩展能力已基本落地
- 项目进入“后 MVP 扩展与工程治理并行”阶段

这一阶段的重点不再是补齐最初骨架，而是：

1. 收紧文档与规范体系
2. 完成尚未收尾的增强项
3. 为下一条大功能线建立可执行任务规格
4. 处理影响后续开发效率的横切工程问题

## 当前优先级

### P1

- 文档体系继续收口，清理命名、职责、归档边界
- 完成热搜 P3

### P2

- 建立聊天 MVP 的正式任务规格
- 明确聊天与平台调解的边界、依赖与最小实现范围

### P3

- 推进平台调解
- 前端包体积与加载性能治理
- 继续补强 API 规范化能力

## 下一阶段入口条件

只有在以下条件满足后，才适合把“聊天/调解”作为主线推进：

- 当前文档体系基本稳定
- 热搜 P3 完成或明确延期
- 评价、推荐、热搜治理相关行为不再频繁返工
- 当前测试与联调资产能支撑继续扩展

## 当前风险

- 文档现实与代码现实容易再次脱节
- 还没有形成稳定的正式 API 规范产出流程
- 聊天与调解一旦同时推进，容易把任务边界再次搅乱
- 前端包体积与管理页增长会持续抬高维护成本

## 使用规则

- 只记录当前阶段与下一阶段相关内容
- 已经失去当前指导价值的阶段说明，移入 `../archived/`
- 具体执行步骤统一下沉到 `../../08-tasks/`
