# 当前阶段路线图

本文件只记录“当前阶段正在做什么、下一步做什么、当前风险是什么”。

它不是项目历史总结，也不负责记录已经完成的早期阶段细节。

## 当前阶段

当前阶段定义为：

- F0-F6 功能路线已按当前仓库范围完成并收口
- 支付升级已完成实现、文档与沙箱验收
- 当前主线从“功能收口”切换为“上线准备 + 后续扩展储备”

这一阶段的重点是：

1. 按 `launch-preparation-roadmap.md` 推进安全、合规、基础设施、部署、监控和验收
2. 保持“上线准备”与“未来功能扩展储备”分离，不把新的产品扩展重新混入当前收尾阶段
3. 如需继续新增产品能力，先创建新的 roadmap/task，而不是复用已归档的功能路线图

**阶段文档**：本文件说明项目当前处在哪个阶段、当前主线是什么、主要风险是什么。
**上线路线**：`launch-preparation-roadmap.md` 提供 L0-L7 上线准备路线与验收事项。

## 当前优先级

### 功能状态

- **当前功能路线**：此前 F0-F6 收口路线已完成并归档，不再把它视为进行中工作。
- **后续扩展**：主题偏好深化、通知偏好、复杂营销、后台批量能力等如需继续推进，应作为新一轮规划项重新立项。

### 上线 P0

- **安全加固**：HTTPS、JWT 密钥、CORS、输入验证、SQL 注入防护、权限回归
- **隐私合规**：隐私政策、用户协议、Cookie 同意、用户同意日志、数据导出、账号删除
- **基础设施与部署**：数据库连接池、日志、健康检查、备份、Docker、Nginx、CI/CD、回滚

### 上线 P1

- **性能容量**：性能基线、Redis 缓存候选、限流、HTTP 缓存、压测
- **监控运维**：指标、告警、日志检索、runbook、应急预案

## 上线入口条件

只有在以下条件满足后，才适合正式上线：

- `launch-preparation-roadmap.md` 中 L1/L2/L3/L5/L7 的 P0 项完成
- 当前准备上线的功能基线与历史收口范围已经过验证，不再依赖单独的现行功能路线图
- 性能测试达标（500 并发用户，p95 <500ms，错误率 <1%）
- 安全扫描无高危漏洞
- 备份恢复流程经过演练验证
- 运维文档和应急预案完整
- 合规性检查通过（ICP 备案如需要）

## 当前风险

- **范围混淆风险**：不要在上线准备阶段夹带新的功能扩展，避免验收口径持续漂移
- **环境一致性风险**：上线前仍需继续验证 HTTPS、部署、回滚、日志、备份恢复与监控闭环
- **性能优化风险**：Redis 缓存失效策略复杂，需要充分测试
- **数据迁移风险**：图片迁移到 OSS 需要无损且可回滚
- **合规性时间**：ICP 备案需要 20-30 个工作日，需提前启动

- 功能任务和上线准备任务分别引用对应路线图，不在本文件展开实施细节
